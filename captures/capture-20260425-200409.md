# Claude Code Haiku 4.5 Response

%%FILL: what chain-of-thought actually does to the computation — is it more compute, different compute, or both%%

Both. CoT adds tokens, so more forward passes (more compute). But it also changes the *conditioning context* — each intermediate token becomes input to later predictions, which changes the inference trajectory. On decoder-only models like Claude, token N sees the CoT trace in context, so later predictions route differently. It's not "the same computation, more steps"; it's "more steps that feed back into the model's state," restructuring what the model has access to at decision time.

%%FILL: why CoT helps on some problem types and hurts on others, with a specific example of each%%

Helps on **multi-hop inference with verifiable steps**: math (7×8+9). Without CoT, the token-level prediction has to compress two operations into one emission; error rates are high. With CoT (write "56, then 56+9=65"), each step is lower-entropy, errors don't cascade. Intermediate steps are checkpointable.

Hurts on **tasks where the solution is intuitive but the reasoning path is ambiguous or post-hoc**: "Write a moving paragraph about loss." Forced CoT ("I will establish tone, then use metaphor, then resolve...") locks the model into a rationalized narrative that constrains associative leaps. The trace becomes a cage, not a ladder.

%%FILL: the difference between visible reasoning (text output) and hidden reasoning (latent / interleaved thinking)%%

Visible (CoT): explicit tokens in the output stream. Auditable, but commits the model to a single path — if that path derails, the answer inherits the error. The model cannot backtrack unseen.

Hidden (extended thinking, o1-style): intermediate "thinking" tokens generated but withheld from output. The model can explore multiple paths, verify, restart, without the user seeing failure modes. Final answer reflects richer internal compute. The computation is deeper; the narrative is polished.

%%FILL: what test-time compute scaling looks like — what does 10x more thinking budget actually buy you%%

On **high-variance solution spaces** (competition math, code synthesis): scaling to 3-5x yields superlinear gains (each new path has a real chance of finding a better solution). Beyond 5x, diminishing returns — you've explored the main branches.

On **low-variance spaces** (fact recall, simple format transforms): 10x thinking gains you almost nothing. The model either knows the answer or doesn't; thinking doesn't generate missing knowledge.

Empirical on o1: ~1.5x improvement at 1x thinking (basic CoT), ~3x at 5x thinking (exploration budget), ~3.5x at 20x (exploration + verification overhead). Ceiling is problem-specific, not universal.

%%FILL: when a model's reasoning trace is faithful to its actual computation vs. post-hoc rationalization%%

Faithful: the trace reflects the actual decision path. Signals: (1) reasoning is trained into the model (o1, extended-thinking models with thinking in the loss), (2) the task is deterministic (math, logic, code), (3) the trace is short enough that drift is minimal. Remove the trace mid-generation and the answer changes or degrades.

Post-hoc rationalization: the model generates plausible explanation decoupled from how it decided. Signals: (1) the answer arrives before the reasoning ("the model just knows"), (2) reasoning is tacked on via prompt ("now explain your answer"), (3) the trace and the answer are orthogonal (says "approach X" but solves via approach Y).

Detector: Does the reasoning causally affect the answer? If yes, likely faithful. If the answer doesn't change when you remove or contradict the reasoning, it's window-dressing.

%%FILL: a prompt design that elicits genuine reasoning vs. one that elicits performative reasoning%%

Genuine reasoning:
```
Problem: [hard, multi-step problem]
Solve this step by step. Show your work. If you find an error, correct it.
```
Works when the problem is genuinely hard, the model is trained to reason (o1, extended-thinking), and reasoning is *necessary*.

Performative reasoning:
```
Let's think step by step about this.
Problem: [easy problem with an obvious answer]
```
The model will produce a trace because you asked, but it's post-hoc narration. The answer was available without the trace.

The diff: ask for reasoning only when the problem requires it. Measure causality — does removing the reasoning scaffold degrade the answer? If not, the reasoning is performance theater.

%%FILL: what self-consistency and best-of-N sampling give you that single-shot CoT doesn't%%

Single-shot CoT: one path, one answer, one error mode.

Best-of-N: sample N times, pick the best (via verifier or confidence score). Works if paths are diverse and some are correct. Buys you robustness but requires a good verifier; costs N× inference.

Self-consistency: sample N, collect all final answers, majority vote. Exploits the fact that correct reasoning paths are higher-probability than incorrect ones — correct answers appear more often across samples. On hard reasoning tasks (AMC, code), this gives ~3-5% absolute gain. On simple tasks, N=1 is optimal (no variance to exploit).

Cost: 10-20x inference for ~5% gain. Useful for offline / high-stakes. Not for interactive.
