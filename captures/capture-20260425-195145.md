# Claude Code 4.7

%%FILL: which has the better learning curve and by how much%%
**Go, by a wide margin** — roughly 3–5x faster to productivity. A competent engineer ships useful Go in a week; Rust's borrow checker, lifetimes, and trait system realistically take 2–3 months before the fight-the-compiler phase ends. Go's spec fits in an afternoon read; Rust's Nomicon exists for a reason.

%%FILL: which produces more maintainable code at 100k+ lines%%
**Rust.** The type system encodes invariants the Go reviewer has to catch by hand — nil-ability, send/sync, ownership, exhaustive matches, error variants. At 100k+ LOC the compiler doing that bookkeeping pays back the upfront cost. Go's `interface{}`/`any` sprawl, nil-pointer landmines, and lack of sum types degrade faster as the codebase ages and contributor count grows. Discord's Read States rewrite (Go → Rust) and Cloudflare's Pingora are the canonical receipts.

%%FILL: which is faster in real-world workloads (cite a benchmark)%%
**Rust.** TechEmpower Round 22 (2024): top Rust frameworks (`actix-web`, `axum` with `may_minihttp`/`xitca`) sit at the top of the composite score; the fastest Go entry (`fasthttp`-based stacks) lands roughly 30–40% below the Rust leaders on plaintext and JSON. Rust's no-GC + zero-cost abstractions + LLVM backend wins tail-latency and throughput-per-core on CPU-bound paths. Discord publicly reported their Go→Rust rewrite eliminated GC-induced latency spikes entirely (every 2 minutes → flat).

%%FILL: which has better tooling (compiler, package manager, formatter, debugger)%%
**Rust.** `cargo` is the best first-party package manager + build system + test runner + doc generator + benchmark harness in any systems language. `rustfmt`, `clippy`, `rust-analyzer` are all best-in-class. Go's tooling is excellent and famously fast (`go build`, `gofmt`, `go test`, `gopls`) — but `cargo` is one tool that does what Go needs five for, with workspaces, features, and a real dependency resolver. Debugger story is a wash: both lean on gdb/lldb/delve with rough edges.

%%FILL: which has better library ecosystem for network services%%
**Go.** This is the inversion. Go was *built* for network services at Google and the ecosystem reflects 15 years of that focus: gRPC's reference implementation, Kubernetes/etcd/Consul/Vault/Docker/Prometheus/Caddy/Traefik — the cloud-native stack is Go. Need an S3 client, an OIDC provider, a Kafka consumer, a service mesh sidecar? The mature, battle-tested option is Go. Rust's `tokio`/`hyper`/`tonic`/`axum` stack is excellent and growing fast, but the breadth and operational maturity gap is real.

%%FILL: which language design decision do you most disagree with for each%%
- **Go: lack of sum types / proper enums.** Errors-as-values without tagged unions means you're back to sentinel comparisons, type switches on `error`, and `errors.Is`/`errors.As` ceremony. Generics arrived (1.18) but the deliberate omission of sum types is the single decision that most degrades large-codebase correctness. The `if err != nil` meme is downstream of this.
- **Rust: async coloring + the executor non-decision.** Shipping `async/await` without a standard executor splits the ecosystem (`tokio` vs `async-std` vs `smol`), forces "what color is your function" discipline, and makes library authors choose runtime-agnostic gymnastics or pick a side. The std/tokio split is the largest unforced complexity tax in modern Rust.

%%FILL: a project type where the loser-overall is actually the right choice%%
**Picking Rust as the "winner overall" here — so the loser is Go. Right choice for Go: a mid-size internal control-plane / orchestrator / CRUD-over-gRPC service with a 4–8 person team and 18-month time-to-value pressure.** Kubernetes operators, internal platform APIs, CI/CD glue, infrastructure daemons. The team ships in week 2 instead of month 3, the GC pause is irrelevant at the SLO, the operational ecosystem (client-go, controller-runtime, OpenTelemetry, Prometheus client) is native, and the marginal perf Rust would buy is invisible next to the network and database it talks to. Choosing Rust here is engineering vanity.
