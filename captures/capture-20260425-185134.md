%%FILL: the literal mathematical operation a single FFN neuron performs%%

A single FFN neuron in a transformer block computes: `a = σ(w_in · x + b_in)`, where `x ∈ ℝ^d_model` is the residual stream vector at that token position, `w_in ∈ ℝ^d_model` is the neuron's input weight row (one row of the up-projection matrix `W_in ∈ ℝ^{d_ff × d_model}`), `b_in` is its scalar bias, and `σ` is the elementwise nonlinearity (GeLU, SwiGLU's gated variant, ReLU in older models). That scalar activation `a` is then scattered back into the residual stream by the neuron's output weight column `w_out ∈ ℝ^d_model` (one column of `W_out ∈ ℝ^{d_model × d_ff}`), contributing `a · w_out` additively. So per neuron: one dot product in, scalar through nonlinearity, one rank-1 outer-product write back. The neuron's "input direction" is `w_in`, its "output direction" is `w_out`, and these are *not* tied — the read and write geometries are independent, which matters for interpretation. In SwiGLU/GLU variants the gate adds a second input projection multiplied elementwise before the down-projection, so the "neuron" is really a (gate, value) pair, but the per-channel structure is the same.

%%FILL: what "polysemantic" means at the neuron level and why it's the rule rather than the exception%%

Polysemantic = a single neuron's activation correlates with multiple, semantically unrelated input features. Neuron 1402 fires on French text, on DNA sequences, on Python list comprehensions, and on images of dogs — not because there's a hidden unifying concept, but because the network has packed unrelated features onto the same channel. It's the rule, not the exception, because the network has more features it wants to track than it has neurons (or residual dimensions) to dedicate one-per-feature. Anthropic's *Toy Models of Superposition* (Elhage et al. 2022) showed this is what optimization pressure produces whenever feature count > dimension count and features are sparse: the loss landscape rewards cramming, accepting a small interference cost in exchange for representing more things. Monosemantic neurons exist mostly when (a) a feature is extremely common and important enough to "earn" a dedicated channel, or (b) the architecture forces it (e.g., sparse autoencoder dictionaries). In a dense FFN of a frontier model, picking a random neuron and asking "what does it mean?" almost always returns a list, not a label.

%%FILL: a specific neuron that has been identified and characterized in published research%%

The cleanest published example is the **"an" neuron** in GPT-2 small, layer 31 of the MLP — characterized in Anthropic's *Softmax Linear Units* / early circuits work and reproduced in TransformerLens tutorials. It activates strongly when the next token should be the indefinite article "an" rather than "a," i.e., when the upcoming noun starts with a vowel sound. Its input weights read from the residual stream features that encode "next token begins with a vowel," and its output weights write a logit boost onto the " an" token (with leading space) and a suppression onto " a." It's an unusually monosemantic neuron — a useful pedagogical specimen precisely because most aren't this clean. Also widely cited: the **base64 neuron** and **French neuron** in GPT-2 medium (fire on long base64 strings and on French text respectively, identified by Conmy et al. and the Neuronpedia catalog), and the **curve detectors** in InceptionV1 from Olah et al.'s original Distill *Circuits* thread — vision rather than LLM, but the canonical "we actually understand this neuron" reference. For frontier-scale work, Anthropic's *Scaling Monosemanticity* (2024) on Claude 3 Sonnet bypasses raw neurons entirely and characterizes SAE *features* — the "Golden Gate Bridge feature," the "code error feature," the "sycophancy feature" — because at frontier scale individual neurons are too polysemantic to name.

%%FILL: how superposition packs more features than dimensions and why this works%%

Superposition stores `n` features in `d` dimensions where `n > d` by assigning each feature a near-orthogonal (but not exactly orthogonal) direction in the d-dimensional space. The Johnson-Lindenstrauss lemma guarantees that in d dimensions you can fit exponentially many vectors with pairwise cosine similarity below ε — roughly `exp(d·ε²)` near-orthogonal directions. So a 12,288-dim residual stream can host millions of feature directions whose mutual interference is small. It works because **features are sparse**: any given token activates only a tiny fraction of the total feature dictionary (the "Golden Gate Bridge" feature is off for 99.999% of tokens). When feature A and feature B have small but nonzero dot product, they only interfere when *both* fire simultaneously — and sparsity makes that collision rare. The network learns to tolerate the residual interference (it's noise, but bounded noise) in exchange for representational capacity. The nonlinearity (GeLU, ReLU) is load-bearing here: it lets the network *clean up* low-magnitude interference by thresholding, recovering near-clean feature reads even though the storage is overlapping. Without nonlinearity, superposition would just be linear projection and the interference would propagate. The Elhage et al. toy model shows the phase transition explicitly: at low sparsity, the model uses one-feature-per-dimension; as sparsity increases past a threshold, it switches to packed superposition geometries (digons, triangles, tetrahedra, pentagons in 2D — uniform polytopes that minimize interference for a given feature count).

%%FILL: what activations look like across a batch — sparse? dense? what distribution?%%

For a typical FFN neuron with GeLU/ReLU: across a large batch of tokens, activations are **heavily sparse and heavy-tailed**. Most tokens (often 80–95%) leave the neuron at or near zero (post-nonlinearity); a small fraction produce moderate activations; a long tail produces large outliers. The distribution is roughly log-normal in the positive tail with a hard or soft floor at zero — what people sometimes call "burst" or "spike-and-slab" activation patterns. Pre-activation (before the nonlinearity) the distribution is closer to Gaussian-ish, centered slightly negative so the nonlinearity zeros most of it. Across the *whole* FFN layer, the *L0 sparsity* (fraction of neurons active per token) varies by architecture: dense ReLU/GeLU FFNs typically have 10–30% of neurons active per token in early/mid layers, sometimes lower in late layers. SwiGLU/GLU variants and MoE layers push this further. Critically, the *which* neurons fire is highly token-dependent — a French token activates a different sparse subset than a Python token. This token-conditional sparsity is exactly the precondition for superposition to work: if every neuron fired on every token, interference would be catastrophic. There's also a small population of **outlier "always-on" neurons** with consistently large activations across all tokens — these often encode positional or layer-norm-bias-like signals and are disproportionately important for model performance (Dettmers et al. on outlier features in quantization).

%%FILL: the difference between "this neuron fires for X" and "X is represented in this direction"%%

"Neuron N fires for X" is a claim about a **basis-aligned channel**: there exists an index N such that activation `a_N` correlates with feature X. This is a privileged-basis claim — it says the network chose to align feature X with the standard basis vector `e_N`. "X is represented in direction `v`" is a **basis-free** claim: there exists some vector `v ∈ ℝ^d` (not necessarily a coordinate axis) such that `v · x` correlates with feature X. The second is strictly more general and almost always more accurate. Neurons can appear polysemantic *because the privileged basis is wrong* — feature X actually lives along direction `v = 0.7·e_N + 0.5·e_M + …`, and looking at neuron N alone gives you a smeared, multi-feature read. The MLP nonlinearity *does* create some basis privilege (GeLU acts elementwise, so the coordinate axes aren't fully arbitrary the way they are in the residual stream — the residual stream has no privileged basis at all, only the MLP and attention head readouts do). But even with that partial privilege, individual features routinely sit on directions that are linear combinations of multiple neurons. This distinction is the whole reason **sparse autoencoders** work: the SAE learns a dictionary of feature directions `{v_1, v_2, …}` in activation space without assuming they're axis-aligned, and recovers monosemantic features from polysemantic neurons. "Neuron N fires for X" is a story about the model's chosen coordinates; "X is in direction v" is a story about the model's geometry. The geometry is what's real; the coordinates are an artifact.

%%FILL: why looking at single neurons was the wrong abstraction for years and what replaced it%%

Looking at single neurons was the wrong abstraction because (1) most neurons are polysemantic, so "what does this neuron do?" has no single answer; (2) features are stored in superposition along non-axis-aligned directions, so the neuron basis is the wrong coordinate system to ask the question in; (3) circuits — the actual computational units — span multiple neurons across multiple layers and attention heads, so per-neuron analysis fragments a coherent computation into incoherent pieces; (4) the residual stream has no privileged basis at all, so any "neuron-like" view of it is arbitrary. What replaced it, in roughly chronological order: **circuits-level analysis** (Olah et al., Distill, 2020) — characterizing small subgraphs of neurons that together implement an algorithm, with curve detectors and dog-head detectors as canonical examples; **the residual stream / induction heads frame** (Elhage, Olsson et al., *A Mathematical Framework for Transformer Circuits*, 2021) — treating attention heads as the unit of analysis and the residual stream as a bandwidth-limited communication channel; **mechanistic interpretability of specific behaviors** (the IOI circuit, Wang et al. 2022; modular addition, Nanda et al. 2023) — reverse-engineering a single capability end-to-end across layers; **sparse autoencoders / dictionary learning** (Bricken et al., *Towards Monosemanticity*, 2023; Templeton et al., *Scaling Monosemanticity*, 2024) — the current frontier-scale answer: train an overcomplete sparse dictionary on activations, recover monosemantic features as dictionary atoms, then characterize *features* rather than neurons. The unit of interpretability in 2026 is the SAE feature (or transcoder feature, or attribution-graph node), not the neuron. The neuron persists as an implementation detail of how the network *stores* features, not as the level at which features *exist*.
